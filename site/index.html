02 / 工作流

从图集到桌面，
只有三个动作。

01
⌗

导入工程

选择 pet.json，应用读取同目录的大图集和 Studio 配置。

02
⌁

编排动作

逐帧微调，设置播放次数与优先级，再为动作选择触发事件。

03
↗

放到桌面

开启工程显示；每只桌宠独立运行、移动并响应自己的事件。

03 / 工程格式

透明、可读，
没有隐藏的素材库。

导出的工程就是 JSON 配置加一张完整 PNG，易于备份，也方便继续交给其他工具。

Codex v2 compatible

保留简单，才能保持自由。

pet.json 描述身份与图集位置，spritesheet.png 保存全部画面， studio.json 则记录更丰富的动作和触发逻辑。

查看格式文档 ↗ 
my-pet/
├── pet.json          # 身份与图集路径
├── spritesheet.png   # 唯一完整图集
└── studio.json       # 动作与事件逻辑
