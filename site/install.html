01 / Put it in the right place

Three steps make it
a searchable Mac app.

Downloads is only a holding area. macOS adds third-party apps to its app browser and Spotlight after they are placed in Applications.

In Finder, choose Go → Applications (⇧⌘A), then drag SpritePetStudio.app into that folder.

Launch it once from Applications. It can then appear in Spotlight and the macOS Apps view; drag its icon to the Dock if you want it pinned.

⌘

Check that the exact path is /Applications/SpritePetStudio.app—not the ZIP and not an app still inside Downloads. Then press ⌘ Space and search for SpritePetStudio.

02 / First launch

The warning is expected.
The source still matters.

The current open-source build is ad-hoc signed and has not been notarized with an Apple Developer ID. macOS therefore cannot verify its developer automatically.

!
macOS Gatekeeper

Open it through Privacy & Security

  1. 1

    Try to open SpritePetStudio from Applications once. In the warning shown above, click Done—not Move to Trash.

  2. 2

    Open System Settings → Privacy & Security, then scroll down to the Security section.

  3. 3

    Find the message about SpritePetStudio and click Open Anyway. Apple keeps this button available for about one hour after the failed launch.

  4. 4

    Authenticate with your password or Touch ID, then confirm Open. macOS saves this app as an exception.

Before you override

Verify the download.

Download both release files into the same folder, then run this command in Terminal. Continue only when it prints “OK”.

shasum -a 256 -c SpritePetStudio-macOS.zip.sha256

Why this happens

Not a Finder problem.

The alert is Gatekeeper reacting to an unnotarized build. Moving the app to Applications solves indexing; approving it in Privacy & Security solves first launch.

03 / Troubleshooting

Still stuck?
Check these first.

“Open Anyway” is missing

Try launching the app from Applications again, dismiss the warning, and immediately reopen Privacy & Security. On a managed work or school Mac, administrator policy may remove this option.

The app is not in Spotlight

Confirm the app is directly inside Applications, open it once from Finder, then wait a moment and search its exact name. A ZIP or an app left in Downloads is not installed.

Advanced fallback for a verified release

Only after verifying the official release checksum, the commands below remove the quarantine flag from this one app and open it. Never use this on an untrusted download.

xattr -dr com.apple.quarantine "/Applications/SpritePetStudio.app"
open "/Applications/SpritePetStudio.app"
